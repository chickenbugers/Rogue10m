# Rogue10m 메인 UI 기획안

## 참고 방향

- 기본 레이아웃은 `기획서/메인UI초안.pptx`와 로스트아크 HUD 이미지를 기준으로 한다.
- 1인칭 로그라이크이므로 전투 중 직접 조작이 필요한 HUD는 화면 하단에 밀도 있게 배치한다.
- 장비, 아이템, 스킬, 설정 패널은 화면 버튼 클릭보다 단축키 호출을 기본으로 한다.
- 모든 UI는 화면 비율 기반으로 배치하되, 작은 해상도에서 깨지지 않도록 최소 크기를 둔다.

## 메인 HUD 구성

### 시스템 로그

- 위치: 좌측 하단.
- 표현: 반투명 패널.
- 내용: 아이템 획득, 경험치 획득, 시스템 메시지, 주요 전투 결과.
- 표시 줄 수를 초과하면 스크롤바를 보여준다.
- 현재 구현은 최신 로그 중심 표시이며, 이후 마우스 휠/키보드 스크롤 입력을 연결한다.

### 레벨 및 경험치

- 위치: 화면 최하단 좌측.
- 레벨 표시와 경험치 바를 분리해서 보여준다.
- 경험치를 얻으면 PlayerState의 경험치 비율을 받아 경험치 바가 채워진다.

### 몬스터 정보

- 위치: 화면 상단 중앙.
- 출력 정보: 몬스터 이름, 레벨, 속성, 상태, 현재 체력, 최대 체력, 체력 비율.
- 현재 몬스터는 기본 레벨/속성 값을 직접 가진다.
- 이후 몬스터 Data Asset으로 레벨, 속성, 등급, 약점 정보를 이전한다.

### 스킬 영역

- 위치: 하단 중앙 좌측.
- 좌클릭, 우클릭, 점프 공격, 차징 공격 등 전투 입력에 대응하는 슬롯을 보여준다.
- 슬롯 수는 장착 무기와 숙련도/해금 상태에 따라 늘어나는 구조로 설계한다.
- 아직 연결되지 않은 슬롯은 잠금 상태로 표시한다.
- 각 슬롯은 아이콘, 단축키, 쿨타임 숫자, 쿨타임 오버레이를 가진다.

### 체력바

- 위치: 스킬 영역 위.
- 현재 체력과 최대 체력을 함께 표시한다.
- 전투 중 가장 자주 봐야 하므로 스킬 슬롯과 가까운 곳에 둔다.

### 스테미나 바

- 위치: 사용 아이템 영역 위.
- 현재 스테미나와 최대 스테미나를 함께 표시한다.
- 마나가 활성화된 경우 같은 영역에 추가 자원 바를 표시한다.

### 아이덴티티

- 위치: 하단 중앙.
- 현재는 고정 시스템이 없으므로 무기 숙련도와 특수 자원 게이지 자리만 제공한다.
- 이후 무기별 전용 자원, 캐릭터 고유 자원, 변신/폭주/집중 같은 전투 특화 요소를 연결한다.

### 아이덴티티 바

- 위치: 아이덴티티 창 하단.
- 마나, 기력, 무기 전용 게이지 등 특수 자원을 표시할 수 있도록 둔다.
- 현재는 PlayerState의 Identity 값을 표시한다.

### 아이템 획득 창

- 위치: 화면 중앙 우측.
- 표현: 거의 투명한 알림 리스트.
- 아이템/경험치 획득 로그가 들어오면 최근 획득 정보가 상단에 추가된다.
- 각 알림은 3초 뒤 제거된다.
- 제거 후 아래 알림은 자연스럽게 위로 올라오는 리스트 구조로 관리한다.

### 미니맵

- 위치: 우측 상단.
- 현재 맵과 캐릭터 위치를 2D 상단 시점처럼 표현한다.
- 플레이어, 몬스터, NPC, 목표 지점은 서로 다른 색상의 아이콘으로 표시한다.
- 이후 실제 월드 좌표, 몬스터/NPC 탐색 결과, 맵 Data Asset을 연결한다.

## 단축키

- 장비창: `I`
- 아이템 창: `B`
- 스킬트리: `K`
- 설정: `O`

## 추가 확인 필요

- 사용자가 제공한 유튜브 링크는 현재 작업 환경에서 직접 열람이 되지 않았다.
- 영상 확인이 가능해지면 쿠르잔 전선 전투 중 HUD 밀도, 퀘스트/미니맵 정보량, 하단 스킬 슬롯 간격, 아이템 알림 연출을 추가로 비교한다.
