## 작업 요약

- 장비창 슬롯 구성을 현재 기획 기준에 맞춰 재정리했다.
- 장비창 왼쪽에는 `주무기`, `머리`, `갑옷`, `신발`을 배치했다.
- 장비창 오른쪽에는 `보조무기`, `반지`, `귀걸이`를 배치했다.
- 샘플 아이템과 장비 설명을 한국어로 변경했다.
- 아이템 툴팁이 `아이템명`과 `아이템 세부 설명`을 분리해서 보여주도록 정리했다.
- 기본 공격 로그에 공격 종류와 예상 피해량이 표시되도록 했다.

- 몬스터가 피해를 받으면 몬스터 위치 위에 떠오르는 피해 숫자가 표시되도록 했다.
- 플레이어가 피해를 받으면 짧은 붉은 화면 피드백이 표시되도록 했다.
- 주요 전투 디버그 문구와 장비/아이템 UI 문구를 한국어로 변경했다.
- 향후 개발 규칙으로 `PlayerState`, 기능별 `Component`, `Data Asset` 우선 사용 방향을 문서화했다.

- 장비창에서 장착된 아이템을 좌클릭 드래그로 잡을 수 있도록 HUD 드래그 상태를 확장했다.

- 공격 자원 타입 enum은 `Health`, `Stamina`, `Mana`, `Energy`를 지원하도록 준비했다.
- 신규 공격 Data Asset은 기본적으로 체력과 스테미나 비용 항목을 가진다.
- 공격 실행 전 자원 비용을 확인하고, 체력/스테미나/마나 비용이 부족하면 공격을 실행하지 않도록 준비했다.
- 기력은 향후 전용 컴포넌트가 생기면 연결할 예약 값으로 두었다.

- 기본 캐릭터가 즉시 사용할 수 있는 공격을 일반 좌클릭과 점프 좌클릭으로 제한했다.
- `BP_FirstPersonCharacter`에는 `DA_Attack_Unarmed_Primary`, `DA_Attack_Unarmed_JumpPrimary`만 기본 공격 Data Asset으로 연결했다.
- 우클릭, 점프 우클릭, 차징 좌클릭, 차징 우클릭 슬롯은 비워 두어 Data Asset이 없으면 입력이 잠기는 흐름을 확인할 수 있게 했다.
- 스킬트리의 `Knuckle` 표시명을 `권`으로 변경했다.
- 무기 선택 화면에서 권을 가장 앞에 배치하고, 단검/대검/쌍검/활/지팡이는 잠김 상태로 표시했다.
- 권 숙련도는 기본 1, 다른 무기 숙련도는 0으로 표시한다.
- 권 스킬트리 내부에는 커맨드 왼쪽, 스킬 오브젝트 오른쪽 구조로 기본 권격, 도약 권격, 연속 권격 2, 연속 권격 3을 배치했다.
- 기본 제공 스킬인 기본 권격과 도약 권격은 습득됨 상태로 두었다.
- 연속 권격 2와 연속 권격 3은 테스트를 위해 앞 스킬이 해금되면 클릭으로 다음 스킬을 해금할 수 있도록 준비했다.
- 아이템/문서 표기에서도 너클 명칭을 권 기준으로 맞췄다.

## Notion 정리 후보: 권 스킬트리 해금 규칙

- 무기 숙련도는 0부터 10까지 관리한다.
- 스킬트리가 해금되는 순간 해당 무기의 숙련도는 1이 된다.
- 현재 기본 무기 숙련도는 권 1이며, 나머지 무기는 0으로 잠긴다.
- 무기를 획득하거나 기본 스킬서 또는 NPC에게 스킬을 배우면 해당 무기 스킬트리에 접근할 수 있게 확장한다.
- 권 스킬트리는 기본 좌클릭과 점프 좌클릭을 시작점으로 두고, 이후 콤보 스킬은 앞 스킬 습득 여부를 조건으로 해금한다.

## 추가 작업: 권 스킬트리 라인형 구성과 실제 사용 잠금

- 권 스킬트리를 커맨드 박스와 오른쪽 스킬 노드가 이어지는 라인형 구조로 재구성했다.
- 좌클릭 라인은 `기본 권격 -> 연속 권격 2 -> 연속 권격 3 -> 권 마무리` 흐름으로 배치했다.
- 점프 좌클릭 라인은 `도약 권격 -> 공중 연계 -> 낙하 권격` 흐름으로 배치했다.
- 기본 제공 스킬인 기본 권격과 도약 권격은 처음부터 습득됨 상태로 유지했다.
- 나머지 노드는 앞 노드가 해금되기 전까지 클릭할 수 없고, 테스트를 위해 앞 노드가 열리면 다음 노드를 클릭해 해금할 수 있게 했다.
- 공격 실행 시 HUD의 스킬트리 해금 상태를 확인하도록 연결했다.
- 스킬트리에서 잠긴 공격은 Data Asset이 캐릭터 슬롯이나 콤보 체인에 연결되어 있어도 실제 실행되지 않고 잠김 로그를 표시한다.

## Notion 정리 후보: 스킬트리 기반 공격 사용 제한

- 스킬 Data Asset 연결 여부와 스킬 습득 여부는 분리한다.
- Data Asset은 공격의 수치와 판정을 제공하고, 스킬트리는 해당 공격을 사용할 수 있는지 결정한다.
- 기본 권 스킬은 좌클릭과 점프 좌클릭만 습득 상태다.
- 콤보 공격은 Data Asset의 `Next Combo Skill`로 연결되어 있어도 스킬트리에서 해금되기 전까지 실행되지 않는다.

## 추가 작업: 공격 정보 아이콘 추가

- 기본 권격, 연속 권격, 도약 권격 등 각 노드가 짧은 임시 아이콘 라벨과 색상으로 구분되도록 구성했다.
- `Scripts/Editor/CreateDefaultUnarmedAttackDataAssets.py`를 갱신해 기본 주먹 공격 Data Asset에 아이콘 라벨과 색상을 자동 입력하도록 했다.

- 공격 UI는 현재 표시할 공격 Data Asset의 아이콘, 이름, 사용 가능 상태, 남은 쿨타임을 표시한다.
- 실제 아이콘 텍스처가 없는 경우 `Icon Label`과 `Icon Tint` 기반의 임시 아이콘을 보여준다.

## Notion 정리 후보: 콤보 공유 쿨타임

## Notion 정리 후보: 캐릭터 데이터 소유 구조

- PlayerState는 플레이어 지속 상태를 관리한다.
- CombatComponent는 공격 입력 상태와 공격 Data Asset, 콤보, 쿨타임을 관리한다.

- Character는 입력과 월드 상호작용의 연결 지점으로 유지한다.
