# 2026-06-30 개발 로그

## 작업: 콤보 쿨타임 시작 시점 변경

- 콤보 공격의 쿨타임이 공격 즉시 시작되지 않도록 변경했다.
- 콤보가 이어질 수 있는 공격은 `Combo Window Close Seconds`가 끝난 뒤 공유 쿨타임이 시작되도록 조정했다.
- 콤보의 마지막 공격처럼 다음 콤보가 없는 공격은 해당 공격 실행 시점을 기준으로 쿨타임이 바로 시작된다.
- 콤보 입력 가능 시간 안에 올바른 다음 입력이 들어오지 않으면 기본 공격을 새로 시작하지 않고, 콤보 입력 대기 상태가 끝난 뒤 첫 공격 상태와 쿨타임 상태로 돌아가게 했다.
- HUD의 공격 쿨타임 표시는 쿨타임 시작 전 콤보 대기 상태에서는 다음 콤보 스킬 아이콘을 보여주고, 쿨타임이 실제로 시작된 뒤 남은 시간을 표시한다.
- `Scripts/BuildEditor.ps1` 에디터 타겟 빌드를 실행해 컴파일 성공을 확인했다.

## 작업: 무기별 Data Asset 폴더 준비

- 공격 스킬 Data Asset을 무기별로 분리할 수 있도록 폴더 구조를 추가했다.
- 추가한 공격 스킬 폴더는 권, 단검, 대검, 쌍검, 활, 지팡이 기준이다.
- 아이템, 몬스터, NPC, 레벨 Data Asset도 분리할 수 있도록 기본 폴더를 준비했다.

## Notion 정리 후보: 콤보 쿨타임 규칙

- 콤보 공격은 하나의 공유 쿨타임을 사용한다.
- 공유 쿨타임의 길이는 콤보 루트 스킬을 기준으로 관리한다.
- 단, 쿨타임 시작 시점은 마지막으로 입력할 수 있는 콤보 시간이 끝난 뒤로 미룬다.
- 다음 콤보 입력이 성공하면 새 공격의 입력 가능 시간이 다시 열리고, 최종 공격 또는 입력 실패 시 첫 공격 상태로 돌아간다.

## Notion 정리 후보: Data Asset 폴더 구조

- 공격 스킬: `Content/DataAsset/AttackSkill/{WeaponType}`
- 아이템: `Content/DataAsset/Item`
- 몬스터: `Content/DataAsset/Monster`
- NPC: `Content/DataAsset/NPC`
- 레벨/맵 데이터: `Content/DataAsset/Level`

## 작업: 하단 전투 HUD 레이아웃 개편

- 로스트아크식 하단 UI를 참고해 체력과 스테미나 바를 화면 하단 중앙으로 이동했다.
- 체력과 스테미나는 긴 가로 바 형태로 제공하고, 현재값과 최대값을 함께 표시한다.
- 마나는 활성화되어 있을 때만 동일한 하단 바 영역에 추가 표시되도록 유지했다.
- 하단 중앙에 임시 `아이덴티티` 게이지 영역을 추가했다.
- 퀵 슬롯 영역을 `스킬` 공간과 `사용 아이템` 공간으로 시각적으로 분리했다.
- 현재 입력 바인딩을 유지하기 위해 스킬 슬롯은 1~4번, 사용 아이템 슬롯은 5번 키 기준으로 표시했다.

## Notion 정리 후보: 하단 전투 HUD

- 하단 중앙 HUD는 생존 정보, 아이덴티티, 스킬, 사용 아이템을 한 영역에서 관리한다.
- 체력과 스테미나는 항상 표시하고, 마나 또는 기력 계열은 활성화 변수에 따라 표시한다.
- 아이덴티티는 무기 숙련도 또는 무기별 전용 자원을 표현하는 중앙 게이지로 확장한다.
- 스킬 슬롯과 사용 아이템 슬롯은 입력 체계와 데이터 소스가 다르므로 UI에서도 구획을 분리한다.

## 작업: 아이덴티티, 레벨/경험치, 미니맵, 아이템 월드 드랍 원형

- `PlayerState`에 플레이어 레벨, 현재 경험치, 다음 레벨 필요 경험치, 아이덴티티 게이지 기본 데이터를 추가했다.
- 경험치를 얻으면 현재 경험치가 누적되고, 필요 경험치를 넘으면 레벨이 상승하며 다음 요구 경험치가 증가하는 기본 성장식을 준비했다.
- 무기 숙련도는 현재 테스트 단계에서 권/맨손만 1레벨로 열리도록 두고, 다른 무기는 잠긴 상태의 0레벨로 반환하도록 구성했다.
- 하단 중앙 HUD에 레벨, 경험치 바, 아이덴티티 게이지, 현재 무기 숙련도 텍스트를 표시하도록 확장했다.
- 우측 상단에 프로토타입 미니맵을 추가했다. 현재는 실제 맵 캡처가 아니라 위치 좌표, 방향, 테스트 마커를 표시하는 HUD 기반 미니맵이다.
- 인벤토리 또는 장비창에서 드래그 중인 아이템을 두 창 밖으로 놓으면 인벤토리/장비 슬롯에서 제거하고 캐릭터 앞 월드에 드랍 액터가 생성되도록 했다.
- `ARogue10mDroppedItem` 액터를 추가해 드랍된 아이템의 이름 텍스트와 임시 메시가 월드에 보이도록 했다.
- 공격 스킬 Data Asset을 무기별로 분리하기 위한 폴더 구조를 `Content/DataAsset/AttackSkill/{Knuckle,Dagger,GreatSword,DualBlades,Bow,Staff}` 형태로 준비했다.

- 성장 정보는 `PlayerState`에서 관리하고 HUD는 이를 읽어 레벨, 경험치, 아이덴티티, 무기 숙련도를 표시한다.

## 작업: Windows 데스크탑 / Android 모바일 플랫폼 모드 준비

- Windows 데스크탑과 Android 모바일을 분리해서 관리할 수 있도록 플랫폼별 Config 오버레이를 추가했다.
- Windows 설정은 `Config/Windows/WindowsEngine.ini`, `Config/Windows/WindowsInput.ini`에 두고 데스크탑 고품질 렌더링과 마우스/키보드 중심 입력을 기준으로 잡았다.
- Android 설정은 `Config/Android/AndroidEngine.ini`, `Config/Android/AndroidInput.ini`에 두고 모바일 확장성 렌더링, 가로 모드, 터치 인터페이스 사용을 기준으로 잡았다.
- `URogue10mPlatformModeLibrary`를 추가해 C++과 Blueprint에서 현재 실행 모드가 Windows 데스크탑인지 Android 모바일인지 확인할 수 있게 했다.
- Windows 패키징용 `Scripts/PackageWindows.ps1`과 Android 패키징용 `Scripts/PackageAndroid.ps1`을 추가했다.
- `Docs/PlatformBuildModes.md`에 플랫폼별 대상, 설정 파일, 패키징 명령, 앞으로 연결할 모바일/데스크탑 UI 항목을 정리했다.

- 플랫폼 구분은 `URogue10mPlatformModeLibrary`에서 처리하고, HUD와 입력 UI는 해당 값을 읽어 데스크탑/모바일 UI를 분기한다.
- Windows 데스크탑은 키보드/마우스, 고품질 그래픽, 커서 잠금 기반 플레이를 기본값으로 둔다.

## 작업: HUD 공통 UI 폰트 적용

- 게임 UI에 어울리는 무난한 기본 폰트로 엔진 기본 `DroidSansFallback.ttf`를 HUD 공통 폰트로 적용했다.
- `ARogue10mHUD::DrawText` 래퍼를 추가해 기존 HUD 텍스트 출력이 모두 같은 Slate 폰트 경로를 사용하도록 통일했다.
- 한글 UI 텍스트가 안정적으로 표시되도록 한글 지원 폰트를 기본값으로 사용했다.
- 매 프레임 폰트 객체가 반복 생성되지 않도록 CompositeFont 캐시를 추가했다.
- `Rogue10m.Build.cs`에 `SlateCore` 의존성을 추가해 Slate 폰트 기반 Canvas 텍스트 출력이 가능하도록 했다.

## 작업: 공격점 에임 Cross Line 추가

- 1인칭 전투 중 공격 지점을 확인할 수 있도록 화면 중앙에 에임 cross line을 추가했다.
- 중앙 타격점은 작게 비워두고 상하좌우 선을 배치해 실제 공격 지점을 가리지 않도록 구성했다.
- 인벤토리, 장비창, 스킬트리, 설정창처럼 조작을 막는 UI가 열려 있을 때는 조준선이 표시되지 않도록 했다.
- 캐릭터가 사망한 상태에서도 조준선이 숨겨지도록 처리했다.
- 조준선 색상, 길이, 중앙 간격, 두께를 `HUD|Aim` 카테고리의 UPROPERTY로 노출해 이후 Blueprint나 기본값 조정이 가능하도록 했다.

## 작업: FPS 설정 패널 보강

- `GetCurrentFpsLimit`, `SetFpsLimit` 함수를 Blueprint/C++에서 사용할 수 있도록 공개했다.
- 설정 패널의 주요 문구를 한국어로 변경했다.

- 우측 하단 단축키 안내에 `O 설정` 항목을 추가했다.
- 인벤토리, 아이템창, 스킬트리, 설정창이 열릴 때 `GameAndUI` 입력 모드로 전환하고 마우스 클릭/호버 이벤트를 활성화하도록 수정했다.
- 모든 UI 창이 닫히면 `GameOnly` 입력 모드로 돌아가도록 했다.

## 2026-06-30 로스트아크 참조 메인 HUD 개선

### 작업 배경
- 추가 제공된 로스트아크 UI 이미지를 참고해 메인 HUD의 정보 밀도와 구역 구분을 개선했다.
- 1인칭 로그라이크에 맞춰 장비, 아이템, 스킬, 설정은 직접 클릭 버튼보다 단축키 힌트로 유지하는 방향으로 정리했다.

### 구현 내용
- 시스템 로그를 더 넓은 반투명 좌하단 패널로 조정하고, 표시 줄 수를 넘으면 스크롤바가 보이도록 했다.
- 로그 저장 개수를 늘려 아이템/경험치/시스템 메시지를 더 오래 확인할 수 있게 했다.
- 아이템/경험치 획득성 로그가 들어오면 3초 동안 중앙 우측에 획득 알림으로 표시되도록 임시 큐를 추가했다.
- 체력바는 스킬 영역 위, 스테미나/마나 바는 사용 아이템 영역 위로 분리했다.
- 아이덴티티 패널을 하단 중앙에 배치하고, 현재 무기 숙련도와 특수 자원 게이지를 표시하도록 정리했다.
- 몬스터 정보 UI에 이름, 레벨, 속성, 상태, 체력 수치, 체력 퍼센트를 표시하도록 확장했다.
- 기본 몬스터 클래스에 UI 표시용 레벨과 속성 텍스트를 추가했다.
- 미니맵에 플레이어 외 몬스터/NPC/목표 지점용 색상 마커와 간단한 범례를 추가했다.
- 스킬 슬롯은 현재 무기 숙련도/해금 구조를 고려해 표시 개수를 계산하고, 데이터가 없는 칸은 잠금 슬롯으로 보여주도록 했다.
- `기획서/메인UI기획안.md`에 메인 HUD 기획안을 정리했다.
