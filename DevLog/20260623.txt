추가 로그 - 기본 1인칭 런 플로우 구현

요청 내용
- 게임은 기본적으로 아무 무기도 들지 않은 캐릭터로 시작한다.
- 추후 캐릭터 커스터마이징 가능성을 열어둔다.
- 캐릭터는 검, 지팡이, 활 같은 기본 무기를 획득하고 숙련도나 스킬로 강화한다.
- 보스를 잡고 아이템을 얻는 게임 구조를 목표로 한다.
- 1인칭 구조를 유지한다.
- GameMode 설정을 통해 캐릭터가 스폰되고 WASD로 걸을 수 있어야 한다.
- 타이머 UI를 표시하고, 테스트를 위해 게임 시작 30초 뒤 종료 UI를 보여준다.

구현 내용
- ARogue10mGameMode를 직접 기본 GameMode로 사용할 수 있게 abstract를 제거했다.
- Config/DefaultEngine.ini의 GlobalDefaultGameMode를 /Script/Rogue10m.Rogue10mGameMode로 변경했다.
- GameMode에서 기존 First Person 캐릭터 BP와 PlayerController BP를 기본 Pawn/Controller로 지정했다.
- 기존 First Person 템플릿의 Enhanced Input 기반 WASD 이동 구조를 유지했다.
- ARogue10mGameState에 StartRunWithDuration(float)를 추가해 프로토타입 테스트용 런 시간을 지정할 수 있게 했다.
- GameMode에 bUsePrototypeRunDuration과 PrototypeRunDurationSeconds를 추가했다.
- 현재 프로토타입 기본값은 30초로 설정했다.
- ARogue10mHUD C++ 클래스를 추가했다.
- HUD에서 남은 시간, 진행 바, 종료 메시지를 Canvas에 직접 그린다.
- 30초가 지나면 RunPhase가 Defeat로 바뀌고 HUD에 RUN FAILED 메시지가 표시된다.

검증
- UnrealBuildTool 에디터 타깃 빌드 성공.
- 실행 명령:
  - .\Scripts\BuildEditor.ps1 -NoHotReload
- UHT 처리, 컴파일, 링크 모두 성공했다.

Notion 정리 후보: Prototype Run Flow
- 목적: 무기 없는 1인칭 캐릭터로 시작해 런 타이머와 종료 UI를 확인하는 첫 플레이 흐름
- 핵심 클래스:
  - ARogue10mGameMode
  - ARogue10mGameState
  - ARogue10mHUD
- 현재 동작:
  - C++ GameMode가 FirstPerson 캐릭터와 컨트롤러를 지정한다.
  - 기존 템플릿 입력으로 WASD 이동이 가능하다.
  - 프로토타입 런 시간은 30초다.
  - HUD에 타이머와 진행 바가 표시된다.
  - 시간이 끝나면 RUN FAILED UI가 표시된다.
- 추후 확장:
  - 프로토타입 30초 설정을 실제 20분 런으로 되돌리기
  - UMG 기반 정식 HUD 제작
  - 무기 획득 시스템
  - 무기 숙련도와 스킬 성장 시스템
  - 보스 처치와 보상 아이템 루프
  - 커스터마이징 데이터 구조

결정 필요 사항
- 첫 무기는 월드에서 줍는 방식인지, 런 시작 전 선택인지, 튜토리얼로 지급할지 결정 필요.
- 무기 숙련도는 무기 타입별인지, 개별 무기별인지, 캐릭터 전체 성장인지 결정 필요.
- 20분 타이머 종료 시 실패인지, 보스 출현인지, 탈출/정산 단계인지 결정 필요.
- 커스터마이징은 외형 전용인지, 스탯에 영향을 주는지 결정 필요.
- 검, 지팡이, 활의 기본 전투 입력 구조를 공통 액션으로 묶을지 무기별 액션으로 나눌지 결정 필요.

- 플레이 맵은 적당한 사이즈의 오픈 월드로 제작한다.
- 맵 끝까지 약 5분이면 이동 가능한 크기를 목표로 한다.
- 무기 숙련도는 무기 타입별로 진행한다.
- 20분 이내에 보스를 잡으면 클리어, 아니면 실패로 진행한다.
- 커스터마이징은 외형 전용이며 스탯에는 영향이 없다.
- 검, 지팡이, 활 입력은 무기별 액션으로 진행한다.
- 캐릭터 기본 상태는 주먹 공격이 가능한 상태로 유지한다.

확인 내용
- BP_FirstPersonCharacter의 메시 2개는 First Person 템플릿에서 의도된 구조로 판단했다.
- 하나는 월드/3인칭 표현용 메쉬이고, 하나는 소유자만 보는 1인칭 팔/뷰 메쉬 구조다.
- 1인칭 게임과 추후 외형 커스터마이징을 고려하면 현재는 유지하는 편이 좋다.

구현 내용
- ERogue10mWeaponType enum을 추가했다.
  - Unarmed
  - Sword
  - Staff
  - Bow
- ARogue10mCharacter의 기본 EquippedWeaponType을 Unarmed로 설정했다.
- Left Mouse Button 입력으로 DoPrimaryAttack()이 실행되도록 바인딩했다.
- 무기 미장착 상태에서는 DoUnarmedAttack()이 실행된다.
- 주먹 공격은 카메라 전방으로 짧은 Sphere Sweep을 수행한다.
- 명중 시 UGameplayStatics::ApplyDamage로 기본 피해를 적용한다.
- UnarmedDamage, UnarmedRange, UnarmedTraceRadius, UnarmedAttackInterval을 UPROPERTY로 노출했다.
- Docs/WorldStructure.md를 추가해 시작맵, 플레이맵, NPC 역할, 5분 이동 목표 크기를 정리했다.
- Docs/Rogue10m_PrototypePlan.md와 Feature/doc/2026-06-23_prototype-run-flow.md를 최신 설계 결정에 맞게 수정했다.

Notion 정리 후보: World And Combat Starting Rules

  - Item NPC
  - Trait NPC

  - Story NPC

- 클리어/실패:
  - 20분 이내 보스 처치 시 클리어
  - 20분 초과 시 실패
